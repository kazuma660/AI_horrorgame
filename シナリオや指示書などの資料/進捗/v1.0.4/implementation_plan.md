# 実装計画 v1.0.4: fadeBlack詰まりバグ修正・初期シーン画像差し替え

## 1. バグ修正: fadeBlack後に進めなくなる問題

**原因**:
`fadeBlack` エフェクトが発動すると画面全体が黒くなり、テキストボックスが視覚的に隠れる。
テキストボックスは存在しているが見えないため、ユーザーはクリックできる場所があると気づかない。

**修正内容** (`components/Scene.tsx`):
- `<div>` 全体に `onClick={handleScreenClick}` を追加し、**画面のどこをクリックしても次へ進む**ようにした。
- 選択肢表示中はスクリーンクリックを無効化（`if (showChoices) return;`）。
- `fadeBlack` 中は画面下部に `「- クリックして続ける -」` のガイドテキスト（薄く点滅）を表示。

## 2. 画像差し替え: 初期シーン背景

`bed_phone.png` を再生成。
- **旧**: 真っ暗すぎて部屋の雰囲気がわからない画像。
- **新**: スマホの光だけでうっすら部屋の輪郭（天井・壁・家具）が見える日本の6畳間風の一人称視点画像。
