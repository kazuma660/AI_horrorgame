# 画像の再生成とエンディングの実装計画 (v1.0.3)

いただいたご要望に沿って、以下のアップデートを行います！

## 1. 幽霊画像の再生成（より恐ろしく）
現在の画像から「生気を完全に失った死んだような目」「明かりが一切ない真っ暗な空間」を強調した、さらに不気味な覗き穴の画像（`peephole_woman_v2.png`）を生成して差し替えます。

## 2. 最初のシーンが反映されていない件の確認
ファイル自体は設置済みですが、おそらく**ブラウザのキャッシュ（古い画像を記憶する機能）**が原因で表示されていない可能性があります。
画像のファイル名にバージョン情報を付与して強制的に読み込ませるか、ハードリロード（Cmd+Shift+R）で確認できます。

## 3. 画面の「揺れ（Shake）」について
現在のコードでは**「画面全体（背景・エフェクト・テキストボックスすべて）」**が一緒に揺れる仕様になっています。
会話ログ（テキストボックス）も一緒に揺らしたほうがより臨場感が出るか、ご確認いただいてから微調整します。

## 4. 各エンディング専用の背景画像を作成
3種類のエンディング用に、それぞれの結末を決定づける恐ろしい背景画像を生成し、エンディング画面（`app/ending/[id]/page.tsx`）に適用します！

- **【BAD END】**: 抉じ開けられたドアから雪崩れ込んでくる「無数の血走った目」と真っ赤な闇の画像。
- **【TRUE END】**: 薄暗い朝方の部屋の机の上に、ポツンと「自分が寝ている写真」が置かれている不穏な画像。
- **【HIDDEN END】**: 真っ暗な「クローゼットの中」で、スマホの着信画面だけが不気味に青く光っている画像。
